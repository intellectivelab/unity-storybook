# FileUploadDropzone Usage Example

This page provides various examples of using and customizing the FileUploadDropzone component.

<br/>

## Example - Custom dropzone message

This example shows how to replace default dropzone message with a custom one.

### Steps

Implement a wrapper for a FileUploadField and pass property 'dropZoneMessage'. You may also replace a message that shows when dragging is active

```jsx
const CustomFileUploadDropzone = (props) => {
    return <FileUploadField {...props}
                            dropZoneMessage={`Custom message for '${props.label}' field`}
                            dropZoneDragActiveMessage="Drop these PDFs here" />;
};
```

Create mapping conditions for the domain form fields.
```jsx
const isFileDropzone = R.anyPass([
    R.allPass([
        R.propEq("type", "string"),
        R.anyPass([
            R.propEq("ui", "filedroparea"),
            R.propEq("ui", "filedropzone")
        ])
    ]),
    R.allPass([
        R.propEq("type", "file"),
        R.anyPass([
            R.propEq("variant", "droparea")
        ])
    ]),
    R.propEq('type', 'filedroparea'),
    R.propEq('type', 'filedropzone'),
]);

const DomainFieldMapper = R.cond([
    [isFileDropzone, R.always(CustomFileUploadDropzone)],
]);
```

Implement the domain form field factory in the way that the domain form fields mapping comes first.

```jsx
const DomainFormFieldFactory = (props) => DomainFieldMapper(props) || DefaultFormFieldFactory(props)
```
